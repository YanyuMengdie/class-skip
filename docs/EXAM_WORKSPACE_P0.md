# 备考工作台 P0（考试复习一级工作区）

## 用户如何进入

1. 登录后，在顶栏点击 **「考试复习」**。
2. 进入**全屏级**「考试复习 · 备考工作台」（非 `max-w-2xl` 小弹窗）。
3. 顶栏 **「考试」** 仍可打开原有 `ExamHubModal`（兼容旧路径）；**主路径**推荐从「考试复习」进入工作台。

## 「当前考试」意味着什么

- 在工作区顶部的下拉框中选择 **一场**考试作为 **当前备考考试**。
- 选择会写入 **本地 `localStorage`**（键：`examWorkspace_activeExamId_<uid>`，未登录为 `_anon`），刷新页面后仍保留。
- 展示该场考试的标题、日期（若有）、以及 Firestore `examMaterials` 关联材料列表（只读）。

## P0 已做

- 全屏工作台壳 + 返回主学习界面。
- 当前考试持久化（localStorage）。
- **进入本场考试的考前预测**（最小闭环）：
  - 优先使用本场关联材料中**排序后的第一份 `fileHash` 材料**；
  - 若当前主界面已打开同一文件 → 直接打开 `ExamPredictionPanel`（与原先逻辑一致，仍使用当前文档的 `fullPdfText` / LSAP 状态键）；
  - 若未打开 → 触发文件选择器，请在本地选择**与关联材料名称一致的 PDF**；加载成功后自动打开考前预测，**仍停留在备考工作台**；
  - 若本场**仅有云端 `sessionId` 材料** → 尝试 `getUserSessions` + 恢复会话并加载 PDF，成功则打开考前预测，**仍停留在备考工作台**；失败则 **明确提示**，不静默失败。

## P0 明确不做（防范围蔓延）

- 潜力分 / 潜在可拿分 / +N 文案与计算。
- 对话式阐述质量驱动的 BKT 自动上调（保持现有 LSAP/BKT 与面板逻辑）。
- 文科/商科/STEM/艺术等学科代理与 prompt 分支。
- 「今日保温」闪卡流、情境流程心态维度重做。
- **多 PDF 文本合并**进单次考前预测（留给 P1+）。
